# Your own PC Knowledge 📖 based LLM

## Overview

Ever tried to find that one important document buried in your Computer? 😵‍💫 You know, the one with the exact solution you need, hiding somewhere in your 47 different folders, 12 Google Drives, and that USB stick from 2019.

This application runs a fully local Retrieval‑Augmented Generation (RAG) LLM stack that processes documents, creates vector embeddings, and delivers precise answers on your mess.

- Ollama runs the model 🤖
- ChromaDB stores your knowledge 💾
- Chainlit provides the UI 💬
- LangChain glues everything together 🔗

And nope, no API to Antropic or OpenAI at all.

## Setup

- ✅ Ensure you have Python 3.10+ installed.
- 🐙 Ensure you have [Ollama](https://ollama.com/download) installed and pull the `llama3` model:

```bash
ollama pull llama3
```

- Git clone this repository:

```bash
git clone git@github.com:mosesliao/local-kb-llm.git
```

- (Optional) Create and activate a virtual environment:

```bash
python -m venv .venv
# Windows (PowerShell)
.venv\Scripts\Activate.ps1
# macOS / Linux
source .venv/bin/activate
```

- Install dependencies (add any missing packages to `requirements.txt` first):

```bash
pip install -r requirements.txt
```

- Place all the PDF files you want to ingest into the `data` folder.

- Ingest PDFs into Chroma:

```bash
python ingest.py
```

- (Optional) You can ingest documents outside the root folder of this application (example):

```bash
python ingest.py -f C:\Users\<yourname>\iCloudDrive\Documents
```

- Run the app (example):

```bash
chainlit run app.py -w
```

## 🧠 How the RAG Loop Works in This Setup

### Retrieval

ChromaDB performs vector similarity search using embeddings generated by Ollama’s embedding model.

### Generation

Ollama receives a prompt that includes:

- User question
- Retrieved context chunks

### UI

Chainlit handles:

- Chat interface
- Streaming responses
- Session state
